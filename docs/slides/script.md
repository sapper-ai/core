## 1. 도입 (0:00 ~ 1:30) — 슬라이드

### 슬라이드: 타이틀 + 간략 소개

> 안녕하세요. SapperAI 개발자 김규동입니다.
> 본 영상은 조코딩 x OpenAI x Primer AI 해커톤 소개 영상입니다.

> SapperAI는 AI 에이전트의 스킬과 도구 호출을 정적, 동적으로 분석해서 보호하는 프레임워크입니다.
> 정적 분석은 스킬이나 에이전트의 설정 파일을 읽어서, 해킹이 의심되는 패턴이 있는지 규칙 기반으로 검사하는 것입니다.
> 동적 분석은 스킬이나 에이전트를 격리된 환경에서 직접 실행시켜, 출력값에 API 키나 비밀번호 같은 중요 정보가 유출되는지 확인하는 것입니다. 

> 코딩 에이전트와 OpenClaw를 쓰면서 효율성을 느끼는 동시에 보안에 대한 걱정이 들기도 했습니다. 저뿐만 아니라 에이전트를 이용하는 많은 사람들이 같은 필요성을 느낀다고 생각해서 이 프레임워크를 만들었습니다.
> 목표는 oh-my-opencode처럼 설치만 하면 별도 설정 없이 알아서 보안을 처리하는 서비스를 만드는 것입니다.

> 제품 서비스 시연을 진행하겠습니다.

## 3. 기능 상세 — Sapper 비유 (3:30 ~ 6:00) — 슬라이드

### 슬라이드: "Sapper"란?


> 지금부터는 기능을 그림으로 설명드리겠습니다. 
> SapperAI는 Sapper, 즉 군대 공병 중 지뢰병을 모티브로 하고 있습니다. 
> skill과 에이전트에 심은 지뢰같은 위험요소들을 다음과 같은 방법으로 찾습니다.
> 처음에는 지뢰에 해당하는 부품이 있는지 기계적으로 검사합니다. 이것이 정적 분석입니다.
> 그리고 격리된 공간에서 이것이 터지는지 실험하고 터진다고 확인되면 차단하는 것이 동적 분석입니다.
> 다음은 개발이 완료됐고 현재 테스트 중인 기능들입니다.
> 첫 번째는 OpenClaw에 SapperAI를 연동하는 것입니다. OpenClaw가 스킬을 다운로드받고 사용하는 동안 자동으로 검사합니다. 이 기능이 검증되면 OpenClaw 이용자들은 보안 걱정 없이 에이전트를 사용할 수 있게 됩니다.
> 두 번째는 Adversary Campaign, 에이전트 가상 공격입니다. 사용자가 만든 에이전트에 알려진 해킹 기술을 적용해서 보안 대비가 되어있는지 확인합니다. 이 기능이 검증되면, 에이전트를 만들고 → Adversary Campaign으로 테스트하고 → SapperAI 실시간 검증을 켜고 자유롭게 작업하는 흐름이 완성됩니다.
> 현재 제 OpenClaw 환경에서 SapperAI 실시간 안정성 검사를 진행 중이며, 결과가 나오는 대로 영상을 업데이트하겠습니다.

## 4. 시장 + BM (6:00 ~ 7:00) — 슬라이드

### 슬라이드: 고객 + 시장

> 다음은 비즈니스 모델입니다. 저는 코딩 에이전트와 OpenClaw를 이용하는 사람들을 주요 고객으로 삼았습니다.
> 비즈니스 모델은 3단계입니다.
> 처음에는 무료로 최대한 많은 사람들이 이용하도록 합니다.
> 이후 대시보드 같은 편의 기능을 유료로 제공하고, 기능이 고도화되면 엔터프라이즈 기능도 추가할 계획입니다.
> 나중에 코인을 이용한 마이크로 결제가 가능해진다면, 보안 기능 실행 시 자동 결제되는 구조도 만들고 싶습니다. 

> 기존 경쟁사들은 프롬프트 입출력이나 게이트웨이 레이어에서 보호합니다.
> **도구 호출 자체를 전담하는 건 SapperAI뿐입니다.**
>
> Lakera와 비교하면 —
> Lakera는 프롬프트 I/O 보호, 월 8~21달러, 레이턴시 50ms.
> SapperAI는 도구 호출 보호, 무료, 레이턴시 0.002ms.
