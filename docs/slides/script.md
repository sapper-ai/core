## 3. 기능 상세 — Sapper 비유 (2:30 ~ 3:30) — 슬라이드

### 슬라이드: "Sapper"란?

> 여기서 이름 얘기를 잠깐 하겠습니다.
> Sapper는 전장에서 **지뢰를 탐지하고 제거하는 공병**입니다.
> SapperAI도 똑같습니다. AI 에이전트가 걸어가는 길에서 지뢰를 찾아냅니다.

### 슬라이드: 3가지 기능 (그림 설명)

> 기능은 크게 세 가지입니다.
>
> **첫째, 정적 지뢰 분석.**
> `sapper-ai scan` — 설치 전에 MCP 스킬을 미리 스캔합니다.
> 60개 이상의 룰로 패턴 매칭하고, 고위험은 GPT-4.1-mini가 2차 분석합니다.
> 방금 데모에서 본 겁니다.
>
> **둘째, 동적 지뢰 탐지.**
> MCP Proxy — 에이전트와 MCP 서버 사이에 SapperAI가 투명하게 끼어듭니다.
> 코드 변경 0줄. 모든 도구 호출을 실시간으로 검증합니다.
>
> **셋째, 에이전트를 지뢰밭에 보내기.**
> Adversary Campaign — 5가지 공격 시나리오를 4가지 변형으로 자동 생성합니다.
> 총 20개 테스트로 탐지율을 자동으로 검증합니다.

### 슬라이드: 고도화 계획

> 그리고 지금 고도화하고 있는 것.
> **SapperAI in OpenClaw** — OpenClaw 마켓플레이스의 3,000개 이상의 스킬을
> SapperAI로 전수 스캔하는 작업을 진행 중입니다.
> 결과가 나오는 대로 업데이트 영상을 올리겠습니다.

## 4. 시장 + BM (3:30 ~ 4:20) — 슬라이드

### 슬라이드: 고객 + 시장

> 타겟 고객은 두 그룹입니다.
>
> **첫 번째, Claude Code 사용자.**
> MCP 서버와 스킬을 매일 사용하는 개발자들입니다.
> `sapper-ai scan` 한 줄이면 바로 보안 스캔이 됩니다.
>
> **두 번째, OpenClaw 사용자.**
> GitHub 스타 196K의 거대한 생태계입니다.
> 올해 2월, ClawHavoc 사건으로 341개 악성 스킬이 9,000회 이상 설치되는 피해가 발생했습니다.
> 이 두 세계 모두 도구 레이어 보안이 필요합니다.

### 슬라이드: 경쟁사 + 차별점

> 기존 경쟁사들은 프롬프트 입출력이나 게이트웨이 레이어에서 보호합니다.
> **도구 호출 자체를 전담하는 건 SapperAI뿐입니다.**
>
> Lakera와 비교하면 —
> Lakera는 프롬프트 I/O 보호, 월 8~21달러, 레이턴시 50ms.
> SapperAI는 도구 호출 보호, 무료, 레이턴시 0.002ms.
> **1,250배 빠릅니다.**

### 슬라이드: BM

> 비즈니스 모델은 3단계입니다.
> 코어는 MIT 라이선스로 영원히 무료.
> 팀이 커지면 클라우드 대시보드, 기업은 엔터프라이즈 기능으로 수익화합니다.
